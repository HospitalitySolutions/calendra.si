Stripe guest checkout fixes for guest mobile Buy tab
===================================================

Backend
-------
- Guest Stripe checkout still returns through public backend bridge URLs:
  /api/guest/stripe/return and /api/guest/stripe/cancel.
- Added authenticated cancellation endpoint:
  POST /api/guest/orders/{orderId}/checkout/cancel
  This is used by the mobile app when the user presses the phone/browser back button during Stripe checkout.
- Guest orders now reserve the public TenantCode-clientId-counter reference immediately when the GuestOrder is created.
  This means Wallet > Orders no longer needs a temporary ORD-* reference before the invoice exists.
- Stripe CARD checkout no longer creates a Bill/invoice before redirecting to Stripe.
- If Stripe is cancelled/expired/backed out before payment:
  - GuestOrder becomes CANCELLED.
  - Any old unpaid bill created by older code is detached and deleted.
  - No invoice/PDF/folio should exist for the cancelled checkout.
- If Stripe checkout completes successfully:
  - The webhook marks the GuestOrder as PAID.
  - Only then is the tenant Bill/invoice created.
  - The Bill uses the already-reserved TenantCode-clientId-counter reference from the GuestOrder.
  - Entitlement creation and paid receipt generation remain idempotent.

Android guest app
-----------------
- Handles calendra-guest://stripe/return deep links.
- Tracks the active external checkout when opening Stripe/PayPal checkout.
- When the app resumes without receiving a success/cancel deep link, it calls the new cancel endpoint and refreshes Wallet.
- Cancelled orders display as Cancelled / Preklicano.

IOS guest app
-------------
- Handles calendra-guest://stripe/return deep links.
- Tracks active Buy-tab external checkout.
- When the app becomes active again without receiving a Stripe return deep link, it calls the new cancel endpoint and refreshes Wallet.
- Cancelled orders display as Cancelled / Preklicano.

Environment
-----------
Set this in Secrets Manager / production environment:
APP_STRIPE_GUEST_PUBLIC_BASE_URL=https://app.calendra.si

Recommended Platform Admin fallback URLs
----------------------------------------
Checkout success URL:
https://app.calendra.si/api/guest/stripe/return?status=success&orderId={ORDER_ID}&session_id={CHECKOUT_SESSION_ID}

Checkout cancel URL:
https://app.calendra.si/api/guest/stripe/cancel?orderId={ORDER_ID}&session_id={CHECKOUT_SESSION_ID}
