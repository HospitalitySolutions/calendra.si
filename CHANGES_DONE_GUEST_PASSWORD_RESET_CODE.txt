Guest password reset verification-code flow

Changes done:
- Replaced guest password reset email-link flow with a verification-code flow.
- Backend now sends a 6-digit numeric reset code by email instead of mobile/web reset links.
- Reset codes expire after 15 minutes and are stored hashed with BCrypt.
- Added max 5 invalid code attempts; the active reset request is disabled after too many failed attempts.
- Added backend endpoint POST /api/guest/auth/forgot-password/verify-code.
- After a correct code, backend returns a short-lived reset session token that is required by the existing reset-password endpoint.
- Existing reset-password token validation now only accepts tokens after the code has been verified.
- Updated Android guest app forgot-password screen to: email entry -> verification code entry -> new password screen.
- Updated iOS guest app forgot-password screen to: email entry -> verification code entry -> new password screen.
- Updated shared guest-mobile API models, remote API, and repositories for the new verify-code call.
- Updated invalid/expired reset copy from reset-link wording to reset-code/session wording.
