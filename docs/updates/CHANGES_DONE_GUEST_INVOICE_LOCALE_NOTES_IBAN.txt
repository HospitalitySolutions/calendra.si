What changed:
1. Guest invoice language now follows the guest app language at order/checkout time
   - Added an invoiceLocale snapshot to GuestOrder.
   - Guest mobile now sends the current UI locale when creating an order and when confirming checkout.
   - Backend saves this locale on the GuestOrder and uses it before falling back to the guest profile language.
   - Bill PDF generation also falls back to the linked GuestOrder locale/language if the Bill locale is missing.
   - This fixes cases where the guest app UI was Slovenian but the guest profile/server language snapshot was still English.

2. Invoice notes now show only the public Order ID
   - Bank-transfer RF/sklic is still kept inside the UPN QR payload for reconciliation.
   - The visible notes line no longer prints "Notes: Reference: RF...".
   - The visible notes value is now only the public order id, for example "2TEN-3-1".

3. IBAN footer no longer prints the "IBAN:" label
   - The footer renderer suppresses labels for the notes and IBAN footer values.
   - IBAN now renders as just the bank account value, for example "SI56000000000000000".

4. Previous Cards & memberships invoice description fix is preserved
   - Invoice lines still use the purchased ticket/membership/gift-card name instead of the generic transaction service description.

Notes:
- The new GuestOrder.invoiceLocale column is expected to be created automatically if Hibernate ddl-auto=update is enabled.
