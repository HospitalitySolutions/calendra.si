Changes done:

- Fixed Client > Wallet entitlement creation after the wallet purchase invoice is paid.
- Wallet purchase open bills now flush the linked guest order billId when the open bill is converted into a real bill, so later bank-transfer reconciliation / mark-paid events can find the correct guest order.
- The linked guest order payment method is synchronized from the actual closed bill payment method.
- Bill-paid handling is now idempotent: if a paid wallet order already exists but its entitlement was missed, the paid-bill event creates the entitlement safely without duplicates.
- Client > Wallet now self-heals already-paid wallet orders for that client before displaying wallet entitlements, so previously paid wallet purchases can appear after refresh.
- Bank-transfer flow remains payment_pending until the invoice is marked paid/reconciled; entitlement is generated only when paid.
- Normal close-open-bill flow remains unchanged: fiscalization still follows the selected payment method configuration, and bank-transfer/Stripe emailing is handled by the existing checkout-session step used by other open bills.
